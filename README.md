# Investigating the Topic Capacity of Document Embeddings

**AMSI Summer Research Scholarship 2025–26**  
**Author:** Tristan Trieu  
**Supervisor:** Dr Laurence Park, Western Sydney University

## Abstract

This project investigates how multiple topics are represented within document embeddings generated by large language models, with focus on identifying when embedding quality degrades as topic complexity increases. Retrieval performance is used as a quantitative proxy for representational capacity, evaluated using Precision@K, Reciprocal Rank, and Average Precision.

## Research Question

- What is the representational capacity of a single document embedding?
- How does increasing the number of topics within a document affect retrieval effectiveness?

## Experimental Design

1. Start with original single-topic documents
2. Evaluate retrieval accuracy (Original condition)
3. Merge consecutive document pairs to increase topic count (Merge condition)
4. Repeat merging (Merge×2, Merge×3 conditions)
5. Observe accuracy changes across conditions

## Datasets

| Dataset | Documents | Queries | Domain |
|---------|-----------|---------|--------|
| **Cranfield** | 1,400 | 225 | Aeronautical engineering |
| **LISA** | 6,004 | 35 | Library and information science |

## Key Findings

- **Precision@10** declines monotonically across all merging conditions in both datasets
- **Reciprocal Rank** remains relatively stable under initial merging but degrades under heavier compression
- **Average Precision** degrades progressively, becoming statistically significant at Merge×3
- Single-vector embeddings can encode ~1–2 topics before retrieval quality degrades significantly

## Usage

All commands are run from the project root.

### Install dependencies

```bash
pip install -r requirements.txt
```

### Run evaluation

```bash
python main.py cranfield
python main.py lisa
```

### Run paired t-tests

```bash
python run_ttest.py cranfield
python run_ttest.py lisa
```

### Generate figures

```bash
python plot_results.py cranfield
python plot_results.py lisa
python plot_tables.py cranfield
python plot_tables.py lisa
```

### Regenerate embeddings (requires Ollama with nomic-embed-text)

```bash
cd src/cranfield && python original.py
cd src/cranfield && python merge.py
cd src/cranfield && python split.py

cd src/lisa && python original.py
cd src/lisa && python merge.py
cd src/lisa && python split.py
```

## Model

All embeddings are generated using [nomic-embed-text](https://ollama.com/library/nomic-embed-text) (768 dimensions) via Ollama.

## Evaluation Metrics

- **Precision@10**: Proportion of relevant documents in the top 10 results
- **Reciprocal Rank**: Inverse of the rank of the first relevant document
- **Average Precision**: Average of precision values at each relevant document's rank
- **Paired t-test**: Statistical significance testing between Original and each merged condition
